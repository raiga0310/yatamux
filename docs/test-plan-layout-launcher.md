## テスト計画: C-11 アプリ内レイアウトランチャー

## 自動テストで担保する範囲

### TC-C11-07: `.toml` 以外のファイルは候補から除外される
- **種別**: 自動テスト
- **前提**: layouts ディレクトリ相当の一時ディレクトリに `.toml` と `.txt` が混在している
- **操作**: `src/layout_config.rs` のテストで、`LayoutConfig::list_layouts` 相当の `.toml` 列挙ロジックを検証する
- **期待結果**: `.toml` ファイルのベース名のみがソート済みで得られる

### TC-C11-08: layouts ディレクトリが存在しない場合は空扱いになる
- **種別**: 自動テスト
- **前提**: 対象ディレクトリが存在しない
- **操作**: `src/layout_config.rs` のテストで、存在しないディレクトリを読んだ場合の空扱いを検証する
- **期待結果**: パニックせず、空リスト相当の挙動になる

## 手動確認が必要な範囲

### TC-C11-01: Pane モードの `L` キーでランチャーが開く
- **種別**: 手動確認
- **前提**: yatamux 起動済みで Pane モード（`Ctrl+B`）に入っている
- **操作**: `L` キーを押す
- **期待結果**: 画面中央にレイアウト選択ポップアップが表示される

### TC-C11-02: `Esc` / `q` でランチャーを閉じる
- **種別**: 手動確認
- **前提**: ランチャーが表示中
- **操作**: `Esc` または `q` を押す
- **期待結果**: ポップアップが閉じ、Normal モードに戻る

### TC-C11-03: 上下キーで選択が移動する
- **種別**: 手動確認
- **前提**: ランチャーが表示中で複数のレイアウトファイルが存在する
- **操作**: `↑` / `↓` を押す
- **期待結果**: ハイライト行が移動し、端ではクランプされる

### TC-C11-04: Enter で選択中レイアウトが適用される
- **種別**: 手動確認
- **前提**: `%APPDATA%\\yatamux\\layouts\\dev.toml` が存在する
- **操作**: ランチャーで `dev` を選択して `Enter` を押す
- **期待結果**: 既存レイアウトのクローズ完了後、`dev.toml` に基づく新しいレイアウトが適用される

### TC-C11-05: レイアウトファイルが存在しない場合でもランチャーが安全に開く
- **種別**: 手動確認
- **前提**: `%APPDATA%\\yatamux\\layouts\\` にファイルが存在しない
- **操作**: `Ctrl+B` → `L`
- **期待結果**: 空の候補リストとして描画され、クラッシュせずに閉じられる

## 補足

- GUI 側の実際の候補列挙は `crates/client/src/layout/catalog.rs` の `list_available_layouts()` が担当し、`.toml` の列挙とソートに加えてプレビュー生成も行う。
- 現行の自動テストは UI 経路そのものではなく、`LayoutConfig::list_layouts` 相当のファイル列挙ロジックを検証している。ランチャー表示、キー操作、レイアウト切替は手動確認の対象とする。
