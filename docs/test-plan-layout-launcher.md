## テスト計画: C-11 アプリ内レイアウトランチャー

### TC-C11-01: ペインモードの L キーでランチャーが開く
- **前提**: yatamux 起動済み、ペインモード（Ctrl+B）に入っている
- **操作**: `L` キーを押す
- **期待結果**: 画面中央にレイアウト選択ポップアップが表示される

### TC-C11-02: Esc / q でランチャーを閉じる
- **前提**: ランチャーが表示中
- **操作**: `Esc` キーまたは `q` キーを押す
- **期待結果**: ポップアップが消え、Normal モードに戻る

### TC-C11-03: 上下キーで選択が移動する
- **前提**: ランチャーが表示中、複数のレイアウトファイルが存在する
- **操作**: `↓` キーを押す
- **期待結果**: ハイライト行が次のレイアウト名に移動する

### TC-C11-04: 選択が端でクランプされる
- **前提**: ランチャーが表示中、先頭アイテムを選択中
- **操作**: `↑` キーを押す
- **期待結果**: 選択が 0 のまま変わらない（リストの先頭より上にはいかない）

### TC-C11-05: Enter でレイアウトが切り替わる
- **前提**: `%APPDATA%\yatamux\layouts\dev.toml` が存在する
- **操作**: ランチャーで `dev` を選択し Enter を押す
- **期待結果**: 現在のペインが閉じられ、dev.toml のレイアウトで新しいペインが作成される

### TC-C11-06: レイアウトファイルが存在しない場合のメッセージ
- **前提**: `%APPDATA%\yatamux\layouts\` にファイルが存在しない
- **操作**: ランチャーを開く（Ctrl+B → L）
- **期待結果**: ポップアップに「レイアウトファイルが見つかりません」と表示される

### TC-C11-07: list_layouts で .toml 以外のファイルは除外される
- **前提**: layouts ディレクトリに `.toml` ファイルと `.txt` ファイルが混在
- **操作**: `list_layouts()` を呼ぶ
- **期待結果**: `.toml` ファイルのベース名のみ返る（ソート済み）

### TC-C11-08: list_layouts でディレクトリが存在しない場合は空リストを返す
- **前提**: layouts ディレクトリが存在しない
- **操作**: `list_layouts()` を呼ぶ
- **期待結果**: 空のリストが返る（パニックしない）
