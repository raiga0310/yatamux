## テスト計画: ステータスバー + モードベース UI (C-1)

### TC-01: Normal モード表示
- **種別**: 手動確認
- **前提**: 起動直後
- **操作**: 特になし
- **期待結果**: ステータスバーに `[NORMAL]` と `Ctrl+B: ペインモード` ヒントが表示される

### TC-02: Ctrl+B で Pane モードに遷移
- **種別**: 手動確認
- **前提**: Normal モード
- **操作**: `Ctrl+B` を押す
- **期待結果**: ステータスバーが `[PANE]` 表示に切り替わり、PTY に `\x02` は送信されない

### TC-03: Pane モードのヒントが現行キー割り当てを表示する
- **種別**: 実装確認
- **前提**: Pane モード
- **操作**: ステータスバー描画のヒント文字列を確認する
- **期待結果**: `E/O/W/F/X/V/L/S/</>/+/-/q` が表示され、`H/J/K/L` フォーカス移動は表示されない

### TC-04: Pane モードで `E` / `O` が分割を実行する
- **種別**: 手動確認
- **前提**: Pane モード
- **操作**: `E` または `O` を押す
- **期待結果**: それぞれ縦分割 / 横分割が実行され、通常は Normal モードに戻る

### TC-05: Pane モードで `W` がペイン削除を実行する
- **種別**: 手動確認
- **前提**: Pane モード、2 ペイン以上
- **操作**: `W` を押す
- **期待結果**: アクティブペインが削除される

### TC-06: Pane モードで `F` / `X` / `V` / `L` / `S` が各機能を起動する
- **種別**: 手動確認
- **前提**: Pane モード
- **操作**: `F`、`X`、`V`、`L`、`S` をそれぞれ押す
- **期待結果**: `F` はフローティング切替、`X` はスクロールバックの外部エディタ起動、`V` は Copy モード移行、`L` はレイアウトランチャー起動、`S` は保存プロンプト起動になる

### TC-07: Pane モードで `q` / `Escape` がキャンセルになる
- **種別**: 手動確認
- **前提**: Pane モード
- **操作**: `q` または `Escape` を押す
- **期待結果**: Normal モードに戻り、PTY に入力は送られない

### TC-08: Pane モードで `<` / `>` / `+` / `-` により分割比を調整できる
- **種別**: 手動確認
- **前提**: 調整対象の分割を持つレイアウトで Pane モードに入っている
- **操作**: `<` / `>` または `+` / `-` を押す
- **期待結果**: 縦分割比または横分割比が調整され、Pane モードは維持される

### TC-09: ステータスバー分だけペイン描画領域が縮小される
- **種別**: 実装確認
- **前提**: `WM_SIZE` 処理が有効
- **操作**: `content_rect` 計算を確認する
- **期待結果**: クライアント高さから `cell_height * STATUS_BAR_ROWS` が差し引かれた領域がペイン描画に使われる

### TC-10: `skip_char` によりモード切替直後の `WM_CHAR` が抑制される
- **種別**: 自動テスト
- **前提**: `skip_char` の単体テストが利用可能
- **操作**: `skip_char` の set/reset 挙動をテストする
- **期待結果**: `WM_KEYDOWN` でショートカット消費後、対応する `WM_CHAR` が 1 回抑制される

### TC-11: CPU / RAM 使用率が 1 桁でも 2 桁固定で表示される
- **種別**: 自動テスト
- **前提**: ステータスバー右側テキストの整形ヘルパーが利用可能
- **操作**: CPU=9.0、RAM=8.0 で右側テキストを生成する
- **期待結果**: `C:09% M:08%` を含み、`C:9% M:8%` にはならない

### TC-12: CPU / RAM 使用率が 2 桁以上でも整数パーセント表示を維持する
- **種別**: 自動テスト
- **前提**: ステータスバー右側テキストの整形ヘルパーが利用可能
- **操作**: CPU=12.4、RAM=100.0 で右側テキストを生成する
- **期待結果**: `C:12% M:100%` を含み、既存の整数パーセント表示との互換を保つ
