## テスト計画: ステータスバー + モードベース UI (C-1)

### TC-01: Normal モード表示
- **前提**: 起動直後
- **操作**: 特になし
- **期待結果**: ステータスバーに `[NORMAL]` と Ctrl+B ヒントが表示される

### TC-02: Ctrl+B で Pane モードに遷移
- **前提**: Normal モード
- **操作**: `Ctrl+B` を押す
- **期待結果**: ステータスバーが `[PANE]` 表示に切り替わる。PTY に `\x02` は送信されない

### TC-03: Pane モードで H/J/K/L がフォーカス移動
- **前提**: Pane モード、2 ペイン分割済み
- **操作**: `L` を押す
- **期待結果**: フォーカスが隣ペインに移動し、モードが Normal に戻る

### TC-04: Pane モードで E/O が分割
- **前提**: Pane モード
- **操作**: `E` を押す
- **期待結果**: 垂直分割が実行され、モードが Normal に戻る

### TC-05: Pane モードで W がペイン削除
- **前提**: Pane モード、2 ペイン以上
- **操作**: `W` を押す
- **期待結果**: アクティブペインが削除され、モードが Normal に戻る

### TC-06: Pane モードで q / Escape がキャンセル
- **前提**: Pane モード
- **操作**: `q` または `Escape` を押す
- **期待結果**: モードが Normal に戻る。PTY に入力は送られない

### TC-07: Pane モード中の通常文字入力は PTY に届かない
- **前提**: Pane モード
- **操作**: `a` を押す
- **期待結果**: PTY に `a` が送信されない。モードが Normal に戻る（未定義キーは無視）

### TC-08: ステータスバーが右端にペイン番号を表示
- **前提**: 3 ペイン分割、2 番目のペインがアクティブ
- **期待結果**: ステータスバー右端に `pane 2/3` 相当の表示がある

### TC-09: ステータスバー分だけペイン描画領域が縮小される（ユニットテスト）
- **種別**: ユニットテスト
- **操作**: `ClientMode::Normal` 状態で `content_rect` の高さを確認
- **期待結果**: ウィンドウクライアント高さ − `cell_height` がペイン描画高さになる

### TC-10: Ctrl+B を Normal モード中に WM_CHAR が `\x02` を PTY に送らない（ユニットテスト）
- **種別**: ユニットテスト
- **操作**: `should_suppress_char(ClientMode::Pane, 0x02)` を呼ぶ
- **期待結果**: `true`（抑制される）
