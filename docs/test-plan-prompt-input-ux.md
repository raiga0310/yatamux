## テスト計画: 入力プロンプト UX

現行のアプリ内入力欄はレイアウト保存プロンプト（`save_prompt`）のみなので、
まずはここに履歴・Emacs 風編集・ヒント表示を適用する。
将来的に他の入力欄が追加された場合は同じ編集モデルを流用する。

### TC-C39-01: Enter で確定した入力が履歴に追加される
- **種別**: 自動テスト
- **前提**: 空の入力履歴
- **操作**: `foo` を確定し、その後 `bar` を確定する
- **期待結果**: 履歴に `foo`, `bar` が順に保持される

### TC-C39-02: Up / Down で履歴を前後移動できる
- **種別**: 自動テスト
- **前提**: 履歴に複数エントリがある
- **操作**: `Up` を複数回押して過去エントリへ移動し、`Down` で新しい方へ戻る
- **期待結果**: 履歴が新しい順に辿れ、末尾を越えて `Down` した場合は編集中の入力へ戻る

### TC-C39-03: 履歴参照前の編集中入力へ復帰できる
- **種別**: 自動テスト
- **前提**: 編集中の未確定入力がある
- **操作**: `Up` で履歴参照に入り、`Down` で抜ける
- **期待結果**: 履歴参照前の未確定入力がそのまま復元される

### TC-C39-04: `Ctrl+A` / `Ctrl+E` / `Ctrl+B` / `Ctrl+F` でカーソル移動できる
- **種別**: 自動テスト
- **前提**: 文字列が入力済み
- **操作**: 各キーを押す
- **期待結果**: `Ctrl+A` は行頭、`Ctrl+E` は行末、`Ctrl+B` は 1 文字左、`Ctrl+F` は 1 文字右へ移動する

### TC-C39-05: `Ctrl+K` / `Ctrl+U` / `Ctrl+W` / `Ctrl+Y` が kill / yank として動作する
- **種別**: 自動テスト
- **前提**: 文字列が入力済み
- **操作**: `Ctrl+K` / `Ctrl+U` / `Ctrl+W` で削除し、その後 `Ctrl+Y` を押す
- **期待結果**: 削除対象が yank バッファへ保存され、`Ctrl+Y` でカーソル位置へ再挿入される

### TC-C39-06: 入力欄のヒントに履歴と Emacs 風編集キーが表示される
- **種別**: 実装確認
- **前提**: 保存プロンプトが表示中
- **操作**: ヒント行の描画文字列を確認する
- **期待結果**: `Up/Down` による履歴移動と `Ctrl+A/E/B/F/K/U/W/Y` の編集ヒントが表示される

### TC-C39-07: IME 確定文字列が入力欄へ入り、ペインへ誤送信されない
- **種別**: 手動確認
- **前提**: 保存プロンプトを開き、日本語 IME を有効化している
- **操作**: かな漢字変換で文字列を確定する
- **期待結果**: 確定文字列は入力欄へ入り、アクティブペインには送信されない
