## テスト計画: update CLI のエラー経路と再現補助

`yatamux update` は通常は GitHub Releases と実行中インスタンスの IPC に依存する。
不具合再現と integration test を進めやすくするため、まずは失敗時の見え方と
モックしやすさを整える。

### TC-B1-01: 最新版一致時は何も変更せず正常終了する
- **種別**: 手動確認
- **前提**: 実行バージョンと GitHub 最新リリースが一致している
- **操作**: `yatamux update` を実行する
- **期待結果**: `すでに最新バージョンです。` を表示して exit code 0 で終了する

### TC-B1-02: `checksums.txt` ダウンロードの HTTP エラーがそのまま見える
- **種別**: 自動テスト
- **前提**: HTTP ステータス検証ヘルパーが利用可能
- **操作**: `404 Not Found` や `403 Forbidden` を `checksums.txt` ダウンロード失敗として検証する
- **期待結果**: `checksums.txt` と URL と HTTP ステータスを含むエラーになる

### TC-B1-03: バイナリダウンロードの HTTP エラーがそのまま見える
- **種別**: 自動テスト
- **前提**: HTTP ステータス検証ヘルパーが利用可能
- **操作**: `404 Not Found` や `403 Forbidden` をバイナリダウンロード失敗として検証する
- **期待結果**: バイナリ URL と HTTP ステータスを含むエラーになる

### TC-B1-04: IPC 接続失敗時にフォールバック理由が表示される
- **種別**: 実装確認
- **前提**: `ServerConnection::connect()` が失敗するケースを想定できる
- **操作**: `yatamux update` の IPC 接続失敗分岐を確認する
- **期待結果**: 「GUI が見つからない」と決め打ちせず、元の接続エラー内容を表示したうえでヘルパーフォールバックへ進む

### TC-B1-05: 更新 API URL を環境変数で上書きできる
- **種別**: 自動テスト
- **前提**: URL 解決ヘルパーが利用可能
- **操作**: `YATAMUX_UPDATE_API_URL` 未設定 / 設定ありの両方で URL を取得する
- **期待結果**: 未設定時は既定の GitHub Releases API、設定時は上書き値を返す

### TC-B1-06: SaveAndQuit 送信失敗時は helper 起動へ進まず中断する
- **種別**: 実装確認
- **前提**: IPC 接続後に送信側チャネルが閉じる、または書き込みタスクが先に終了するケースを想定できる
- **操作**: `yatamux update` の SaveAndQuit 送信失敗分岐を確認する
- **期待結果**: `SaveAndQuit の送信に失敗` を含むエラーで中断し、どの段階で失敗したか分かる

### TC-B1-07: yatamux ペイン内で IPC 接続に失敗した場合は自己置換へフォールバックしない
- **種別**: 実装確認
- **前提**: `YATAMUX=1` の環境で `ServerConnection::connect()` が失敗するケースを想定できる
- **操作**: `yatamux update` の IPC 接続失敗分岐を確認する
- **期待結果**: 「yatamux ペイン内では自己置換へフォールバックできない」旨を表示して中断し、誤った helper 起動へ進まない
