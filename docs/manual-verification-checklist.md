# 手動検証チェックリスト

自動テスト（`cargo test` / `cargo clippy` / `cargo fmt`）でカバーできない、
実際にアプリを起動して確認が必要な項目をまとめる。

起動コマンド: `cargo run` または `just run`
ログ付き: `$env:RUST_LOG="info"; cargo run`

## C-6: BiDi RTL 制御文字の幅0処理

**ユニットテストで大部分カバー済み。以下は目視確認。**

- [ ] **V-C6-01**: ペイン内で `printf '\u200f\u200eABC'` を実行し、`ABC` の表示位置がずれないこと
  - 期待: RTL マーカー2文字分カーソルが進まず、`ABC` が列0から始まる
- [ ] **V-C6-02**: Arabic 文字列を含む出力（例: `echo مرحبا`）でカーソル位置がずれないこと
  - 期待: テキストは左から始まり後続の文字と重ならない

## C-12: コピーモード

### モード遷移
- [ ] **V-C12-01**: `Ctrl+B` → `V` でステータスバーが `COPY` 表示に切り替わること
- [ ] **V-C12-02**: コピーモード中に `Esc` または `q` で Normal モードに戻ること
- [ ] **V-C12-03**: コピーモードから `q` で抜けた後、Pane モードではなく Normal モードに戻ること

### カーソル移動
- [ ] **V-C12-04**: コピーモードで `h` / `j` / `k` / `l` キーでカーソルが移動すること
  - カーソルは画面上に白いブロック（反転色）として表示される
- [ ] **V-C12-05**: 矢印キー（← ↓ ↑ →）でも同様にカーソルが移動すること
- [ ] **V-C12-06**: カーソルが画面端（行0列0 / 最終行最終列）でクランプされること

### テキスト選択
- [ ] **V-C12-07**: コピーモードで `v` を押すとアンカーが設定され、選択範囲がハイライト表示されること
  - 選択中セルは背景・前景色が反転して表示される
- [ ] **V-C12-08**: アンカー設定後に `j` で下方向へ移動すると選択範囲が広がること
- [ ] **V-C12-09**: 再度 `v` を押すとアンカーが解除され選択が消えること

### クリップボードコピー
- [ ] **V-C12-10**: テキストを選択（`v` + 移動）して `y` を押すとコピーモードが終了すること
- [ ] **V-C12-11**: `y` でコピー後、メモ帳などに貼り付けると選択したテキストが得られること
  - CJK 文字（日本語など）が含まれる場合も正しく貼り付けられること
- [ ] **V-C12-12**: `Enter` キーでも `y` と同様にコピーできること
- [ ] **V-C12-13**: 選択なし（アンカー未設定）で `y` を押してもクラッシュしないこと

### マウスによる選択
- [ ] **V-C12-14**: ペイン内でマウス左ボタンを押しながらドラッグすると選択範囲がハイライトされること
- [ ] **V-C12-15**: ドラッグ後にボタンを離すと選択がクリップボードにコピーされること
- [ ] **V-C12-16**: 複数ペイン環境でペインをまたいでドラッグしても、どちらか1ペインの範囲に限定されること

### 複数ペイン
- [ ] **V-C12-17**: 2ペイン環境でアクティブペインのコピーモード中、非アクティブペインが通常描画されること
- [ ] **V-C12-18**: コピーモード中にフォーカスペインを切り替えると、コピーモードが解除されること（またはアクティブペインで継続すること）

## C-13: `capture-pane` CLI

**前提**: yatamux を起動し、ペインで何か出力させておく（例: `dir` / `ls` 実行済み）。
別のターミナルから以下のコマンドを実行する。

- [ ] **V-C13-01**: 基本動作
  ```powershell
  yatamux capture-pane
  ```
  期待: ペイン0の直近100行がプレーンテキストで標準出力に表示される

- [ ] **V-C13-02**: 行数指定
  ```powershell
  yatamux capture-pane --lines 20
  ```
  期待: 末尾20行のみ表示される

- [ ] **V-C13-03**: ペイン ID 指定（2ペイン環境）
  ```powershell
  yatamux capture-pane --target 1 --lines 50
  ```
  期待: ペイン1（2番目のペイン）の内容が表示される

- [ ] **V-C13-04**: 存在しないペイン ID
  ```powershell
  yatamux capture-pane --target 99
  ```
  期待: エラーメッセージが表示される（クラッシュしない）

- [ ] **V-C13-05**: yatamux 未起動時
  ```powershell
  yatamux capture-pane
  ```
  期待: 「接続できません」旨のエラーが表示される（クラッシュしない）

- [ ] **V-C13-06**: CJK 出力のキャプチャ
  ペイン内で `echo こんにちは` を実行後、`capture-pane` の出力に日本語が含まれること

- [ ] **V-C13-07**: スクロールバック内容の取得
  ペイン内でスクロールバックに流れた内容（画面外）が `--lines 1000` で取得できること

## C-14: `split-pane --dir` CLI

**前提**: yatamux を起動しておく。別のターミナルから実行。

- [ ] **V-C14-01**: 基本動作（作業ディレクトリ指定）
  ```powershell
  yatamux split-pane --dir C:\Users
  ```
  期待: yatamux に新しいペインが縦分割で現れ、`C:\Users` から始まっていること
  （プロンプトに `C:\Users>` が表示される）

- [ ] **V-C14-02**: 水平分割 + ディレクトリ指定
  ```powershell
  yatamux split-pane --dir C:\Windows --direction horizontal
  ```
  期待: 横分割のペインが `C:\Windows` で開く

- [ ] **V-C14-03**: `--dir` 省略時は現在の作業ディレクトリを引き継ぐこと
  ```powershell
  yatamux split-pane
  ```
  期待: 新ペインがサーバープロセスの CWD から始まる

- [ ] **V-C14-04**: 存在しないディレクトリを指定
  ```powershell
  yatamux split-pane --dir C:\nonexistent
  ```
  期待: エラーメッセージが表示される（yatamux がクラッシュしない）

- [ ] **V-C14-05**: ペイン ID 指定で分割元を選択
  ```powershell
  yatamux split-pane --target 0 --dir D:\projects
  ```
  期待: ペイン0が分割され、新ペインが `D:\projects` から始まる

- [ ] **V-C14-06**: `--help` の確認
  ```powershell
  yatamux split-pane --help
  yatamux capture-pane --help
  ```
  期待: 各フラグの説明が表示される
